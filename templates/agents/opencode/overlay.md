## In opencode (and OpenWork)

The loop is available as custom commands under `.opencode/command/` (`spec-orient`, `spec-write`,
`spec-verify`, `spec-sync`, `spec-status`, `spec-commit`). This governance also lives as always-on
project memory in `AGENTS.md` (opencode reads it every session). OpenWork, being powered by
opencode, picks this up automatically. Treat every non-trivial change as gated by the loop above.
