# Specs

Living, per-feature specifications for spec-guard. Each spec answers *what*, *why*, and
*done-when*, and fences its edges (In-Scope / Out-of-Scope / Acceptance / Traceability /
Status). Paired implementation plans live in [`../plans/`](../plans/); formal decisions in
[`../reference/decisions/`](../reference/decisions/).

| Spec | Status |

| [0001 — spec-guard v1](0001-spec-guard-v1.md) | APPROVED |

**Status legend:** `PROPOSED` → `APPROVED` → `IMPLEMENTED` → `SUPERSEDED`.

**Naming:** `NNNN-short-title.md` (or `YYYY-MM-DD-feature.md` in repos that prefer dated specs;
spec-guard's own specs use sequential numbers).
